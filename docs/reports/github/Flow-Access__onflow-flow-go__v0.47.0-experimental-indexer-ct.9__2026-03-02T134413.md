## 한줄 요약
백엔드 테스트 정리 및 확장을 통해 코드 품질을 개선하고 검증을 강화함.

## 운영자 중요 체크
- 테스트 환경에서 백엔드 테스트 실행 결과를 확인하여 정상 작동 여부를 검증한다.
- 기존 테스트 케이스가 정상적으로 유지되고 있음을 확인한다.

## 주요 변경점
- 백엔드 관련 테스트 코드를 정리하고 범위를 확장함.
- 테스트 구조 개선을 통해 유지보수성을 높임.

## RPC/API 영향
- 테스트 개선으로 인해 RPC 서비스의 안정성에 긍정적인 영향을 미칠 것으로 예상됨.
- API 동작에 영향을 주는 변경 사항은 없음.

## 아카이브 노드 영향
- Access 노드의 테스트 개선은 아카이브 노드와 직접적인 영향을 주지 않음.
- 하지만 전체 노드의 안정성 향상은 아카이브 노드 운영에 긍정적인 영향을 미침.

## 운영 액션 아이템
- 테스트 결과를 확인하고, 필요한 경우 로그를 분석하여 문제가 없는지 점검한다.
- 배포 전에 테스트 환경에서 충분한 검증을 수행한다.

## 마이그레이션 체크리스트
- 기존 버전(v0.47.0)에서 새 버전(v0.47.0-experimental-indexer-ct.9)으로 업그레이드 시 별도의 마이그레이션 작업이 필요하지 않음.
- 이미지를 새로 빌드하여 배포한다.

## 위험/주의 사항
